Programación desde Cero con GDScript

Esta guía está pensada para personas que nunca han programado. Incluye ejemplos, errores intencionales y correcciones. El objetivo no es memorizar, sino aprender a pensar.

Enfoque: lógica + hábitos Motor: Godot 4.x Lenguaje: GDScript Incluye: tabla de errores + ejercicios

0) ¿Qué es programar?

Programar es dar instrucciones a una computadora para que haga algo. Un programa es una secuencia de pasos que transforman datos en resultados. En juegos, esos resultados son: movimiento, colisiones, animaciones, puntaje, enemigos, etc.

Lo que siempre existe en un programa

1) Datos (vidas, velocidad, nombre).
2) Decisiones (si está vivo, si chocó).
3) Repeticiones (cada frame, cada segundo).
4) Acciones (mover, saltar, disparar).

Cómo piensa un programador

Divide un problema grande en partes pequeñas: "leer input", "mover", "ver si choca", "actualizar animación". Cada parte se vuelve código.

✓
Meta de esta guía
Que el estudiante pueda leer un error, ubicarlo en el código y corregirlo con seguridad.

1) Variables

Una variable es una "caja con nombre" donde guardas un valor. Ese valor puede cambiar. En un juego: vidas, velocidad, daño, nombre, posición, etc.

¿Qué significa "declarar una variable"?

Declarar es crear la variable y darle un nombre. Luego puedes (o no) asignarle un valor.

Ejemplo: declarar y asignar
extends Node

# ✅ Variables del script (miembros): se pueden declarar fuera de funciones
var vidas = 3
var nombre = "Mario"
var velocidad = 120.0

func _ready() -> void:
	# ✅ Instrucciones ejecutables van dentro de una función
	print(vidas, nombre, velocidad)
Ejemplo: declarar sin valor (luego asignas)
extends Node

# Variable miembro (del script)
var puntaje: int

func _ready() -> void:
	# Aquí ya puedes asignar y usarla
	puntaje = 0
	print(puntaje)

Cómo nombrar una variable (reglas y hábitos)

✅ Descriptiva: velocidad_jugador
✅ minúsculas y _ para separar palabras
❌ No empezar con número: 2vidas
❌ No espacios: vidas jugador
❌ No palabras reservadas: if, for, func

!
Importante (para que no te salga error)
✅ Puedes declarar variables fuera de las funciones (arriba del script): var vidas = 3.
❌ Pero no puedes poner instrucciones que "se ejecutan" en el cuerpo del script, como print, llamados de funciones, if, for, while…
👉 Esas van dentro de una función como _ready(), _process() o _physics_process().
Ejercicio rápido (2–3 min) práctica
  1. Declara una variable llamada monedas con valor 0.
  2. Declara una variable llamada nombre_jugador con tu nombre.
  3. Declara salto_altura con un valor decimal (ej: 250.5).
Revisa que uses el mismo estilo de nombres en todo tu código.

2) Indentación (CRUCIAL después de variables)

En GDScript la indentación (sangría) NO es decoración: define bloques. Si no indentas bien, el programa no entiende qué líneas pertenecen a un if, for, func, etc.

Regla de oro

Todo lo que esté "dentro" de una estructura debe ir indentado.
Godot normalmente indenta con tabs (o espacios) de forma consistente.

!
Consistencia
No mezcles tabs y espacios en el mismo archivo. Eso genera errores confusos.

¿Qué es un bloque?

Un bloque es un grupo de líneas que pertenecen a una instrucción. En GDScript el bloque empieza después de : y se reconoce por la indentación.

✅ Correcto

El print está dentro del if
extends Node

var vidas = 3

func _ready() -> void:
	if vidas > 0:
		print("Sigue jugando")

❌ Incorrecto (falta indentación)

El print quedó fuera del bloque
extends Node

var vidas = 3

func _ready() -> void:
	if vidas > 0:
	print("Sigue jugando")
✕
Cómo lo identificas
Godot marca error en la línea donde esperaba un bloque indentado (justo debajo del if).
Checklist de indentación (para que ellos revisen solos) hábito
  • ¿Cada if/for/while/func termina con :?
  • ¿La línea de abajo tiene sangría (tab/espacios)?
  • ¿Todas las líneas del bloque tienen la misma sangría?
  • ¿No mezclé tabs y espacios?
  • ¿Al cerrar el bloque volví a la sangría anterior?
Mini-reto (5 min): encuentra el error debug
El objetivo es que identifiquen dónde está el error solo mirando la sangría.

Pista: una línea debería estar indentada dentro del segundo if.

3) Tipos de datos

Un "tipo de dato" es qué clase de cosa guarda una variable. No es lo mismo guardar un número que un texto. Si mezclas tipos sin querer, aparecen errores.

Tipos más usados (principiantes)

int (enteros): 0, 1, 2, 3…
float (decimales): 1.5, 120.0…
String (texto): "Mario"
bool (verdadero/falso): true, false

Ejemplos básicos
extends Node

var vidas: int = 3
var velocidad: float = 120.0
var nombre: String = "Mario"
var esta_vivo: bool = true

func _ready() -> void:
	print(vidas, velocidad, nombre, esta_vivo)

Tipos muy usados en Godot

Vector2: posiciones y direcciones 2D (x, y).
Vector2i: igual pero enteros (útil en TileMaps).
Color: colores, transparencia.
Node/CharacterBody2D: objetos del juego.

Ejemplos Godot
extends Node

var posicion: Vector2 = Vector2(100, 200)
var celda: Vector2i = Vector2i(3, 7)
var color: Color = Color(1, 0, 0) # rojo

func _ready() -> void:
	print(posicion, celda, color)
¿Por qué importa el tipo?
Porque determina qué operaciones puedes hacer.
  • Puedes sumar números: 3 + 2
  • Puedes concatenar strings: "Hola" + " Mundo"
  • Pero "Hola" + 3 te obliga a convertir (y puede dar error si no lo haces).

4) Operadores (matemáticos, comparación y lógica)

Los operadores son los "verbos" de la programación: le dicen al programa qué hacer con los datos. Sin operadores no podrías sumar puntaje, comparar vidas, ni decidir si un enemigo debe atacar. Se dividen en cuatro familias que verás en cualquier lenguaje.

4.1 · Operadores matemáticos (aritméticos)

Hacen cálculos numéricos. Son los que más vas a usar para puntaje, movimiento, daño, temporizadores, etc.

Operador Nombre Ejemplo Resultado ¿Para qué sirve en un juego?
+ Suma 10 + 3 13 Sumar monedas, puntaje, experiencia
- Resta 10 - 3 7 Restar vidas, quitar daño, reducir energía
* Multiplicación 10 * 3 30 Escalar velocidad, multiplicar daño crítico
/ División 10 / 3 3 (int) ó 3.33 (float) Repartir botín, calcular promedios
% Módulo (residuo) 10 % 3 1 Saber si un número es par/impar, ciclos de turnos
** Potencia 2 ** 3 8 Escalado exponencial de dificultad
Ejemplo completo: sistema de puntaje
extends Node

var monedas = 10
var bonus = 3
var velocidad: float = 10.0

func _ready() -> void:
	print(monedas + bonus)   # 13  → sumar
	print(monedas - bonus)   # 7   → restar
	print(monedas * bonus)   # 30  → multiplicar
	print(monedas / bonus)   # 3   → dividir (int / int = int)
	print(monedas % bonus)   # 1   → residuo (útil para turnos)
	print(bonus ** 2)        # 9   → potencia

	# Con decimales el resultado cambia:
	print(velocidad / 3.0)   # 3.333...
!
Cuidado con la división entera
Si divides int / int, el resultado es entero (se trunca, no se redondea). Para obtener decimales, al menos uno de los dos debe ser float: 10.0 / 3 → 3.333...

4.2 · Operadores de asignación compuesta (atajos)

Son atajos para modificar una variable usando su propio valor. En vez de escribir vidas = vidas - 1, escribes vidas -= 1. Hacen exactamente lo mismo, pero son más cortos y claros.

Atajo Equivale a Ejemplo Uso típico
+= x = x + valor monedas += 5 Sumar puntaje, recolectar objetos
-= x = x - valor vidas -= 1 Recibir daño, gastar recursos
*= x = x * valor velocidad *= 1.5 Power-ups, multiplicadores
/= x = x / valor velocidad /= 2.0 Ralentizar, debuffs
Ejemplo: power-up de velocidad
extends Node

var velocidad = 100.0
var vidas = 3

func _ready() -> void:
	print(velocidad)    # 100.0

	velocidad *= 1.5    # Agarró un power-up → 150.0
	print(velocidad)

	velocidad /= 2.0    # Le cayó un debuff → 75.0
	print(velocidad)

	vidas -= 1          # Recibió daño → 2
	print(vidas)

4.3 · Operadores de comparación

Comparan dos valores y devuelven true o false. Son la base de todas las decisiones en tu juego: "¿tiene vidas?", "¿chocó?", "¿llegó al límite?".

Operador Significado Ejemplo Resultado ¿Cuándo lo usas?
== ¿Es igual? 3 == 3 true Verificar si el jugador tiene exactamente X vidas
!= ¿Es diferente? 3 != 5 true Verificar si un estado cambió
> ¿Es mayor? 10 > 5 true ¿Tiene suficiente puntaje para pasar de nivel?
< ¿Es menor? 2 < 5 true ¿La vida está baja? ¿Queda poco tiempo?
>= ¿Es mayor o igual? 5 >= 5 true ¿Tiene mínimo X monedas para comprar?
<= ¿Es menor o igual? 0 <= 0 true ¿Se quedó sin vidas? (vidas <= 0)
Ejemplo: sistema de compra en tienda
extends Node

var monedas = 50
const PRECIO_ESPADA = 30
const PRECIO_ARMADURA = 80

func _ready() -> void:
	# ¿Puede comprar la espada?
	print(monedas >= PRECIO_ESPADA)     # true  → sí puede

	# ¿Puede comprar la armadura?
	print(monedas >= PRECIO_ARMADURA)   # false → no le alcanza

	# ¿Ya se quedó sin dinero?
	print(monedas <= 0)                 # false → aún tiene

	# ¿El precio es exactamente 50?
	print(monedas == 50)                 # true
	print(monedas != 50)                 # false

4.4 · Operadores lógicos

Combinan varias condiciones en una sola. Piensa en ellos como conectores: "si tiene llave Y la puerta está cerrada", "si está vivo O tiene revive", "si NO está bloqueado".

Operador Significado Se cumple cuando… Ejemplo en juego
and Y (las dos deben ser verdaderas) Ambas condiciones son true Tiene llave y la puerta está cerca
or O (basta con que una sea verdadera) Al menos una condición es true Está vivo o tiene un revive
not NO (invierte el valor) Convierte true en false y viceversa No está bloqueado → puede moverse

Ejemplo con and

Abrir puerta
extends Node

var tiene_llave = true
var puerta_cerrada = true

func _ready() -> void:
	# Las DOS deben ser true
	if tiene_llave and puerta_cerrada:
		print("Abriendo puerta...")
	else:
		print("No puedes abrir")

Ejemplo con or

Revivir
extends Node

var vidas = 0
var tiene_revive = true

func _ready() -> void:
	# Basta con que UNA sea true
	if vidas > 0 or tiene_revive:
		print("Sigue en el juego")
	else:
		print("Game Over definitivo")
Ejemplo con not y combinaciones
extends Node

var esta_congelado = false
var energia = 50
var vidas = 2

func _ready() -> void:
	# not invierte: si NO está congelado → puede moverse
	if not esta_congelado:
		print("Puede moverse")

	# Combinar todo: puede atacar si tiene energía Y vidas Y no está congelado
	if energia > 0 and vidas > 0 and not esta_congelado:
		print("¡Puede atacar!")
✓
Tabla de verdad rápida
true and true → true · true and false → false · false and false → false
true or true → true · true or false → true · false or false → false
not true → false · not false → true
!
Prioridad de operadores (orden en que se evalúan)
Igual que en matemáticas, algunos operadores se evalúan antes que otros:
1° ** (potencia) → 2° * / % → 3° + - → 4° == != > < >= <= → 5° not → 6° and → 7° or

💡 Si tienes dudas, usa paréntesis para dejar claro el orden: if (vidas > 0) and (energia >= 10):
Error MUY común: = vs == típico

Este es el error más frecuente en principiantes. Se confunden porque se parecen, pero hacen cosas completamente distintas:

= → Asignar: guarda un valor en una variable. "Pon esto aquí".
== → Comparar: pregunta si dos cosas son iguales. "¿Son iguales?"

✅ Asignar (guardar)

var vidas = 3     # Guardar 3 en vidas
vidas = 2         # Cambiar a 2
vidas -= 1        # Ahora vale 1

✅ Comparar (preguntar)

if vidas == 0:       # ¿Vidas es igual a 0?
	print("Game Over")

if nombre != "":     # ¿El nombre NO está vacío?
	print(nombre)
✕
Si escribes if vidas = 0: (un solo =), Godot te dirá:
Invalid assignment in expression — porque dentro de un if necesitas comparar, no asignar.

4.6 · Concatenar Strings (unir textos)

En GDScript, concatenar significa unir varios textos en uno solo usando el operador +. Esto es muy útil para armar mensajes en consola que combinen texto fijo con valores de variables.

String + String

Cuando ambos valores ya son texto, simplemente se unen con +.

Unir dos textos
extends Node

func _ready() -> void:
	var nombre = "Carlos"
	print("Hola " + nombre)
	# Resultado: Hola Carlos

	var arma = "espada"
	print("Equipaste: " + arma)
	# Resultado: Equipaste: espada

String + Número → ❌ Error

No puedes unir un texto con un número directamente. GDScript no sabe cómo mezclarlos.

Esto falla
extends Node

func _ready() -> void:
	var vidas = 3
	print("Vidas: " + vidas)
	# ❌ Error: Invalid operands 'String' and 'int'

La solución: str()

La función str() convierte cualquier valor (número, booleano, etc.) en texto. Así puedes concatenarlo con otros Strings sin errores.

Convertir número a texto con str()
extends Node

func _ready() -> void:
	var vidas = 3
	var nivel = 5
	var es_vip = true

	# Número → texto
	print("Vidas: " + str(vidas))
	# Resultado: Vidas: 3

	# Armar un mensaje más largo
	print("Nivel: " + str(nivel) + " | Vidas: " + str(vidas))
	# Resultado: Nivel: 5 | Vidas: 3

	# Booleano → texto
	print("¿Es VIP? " + str(es_vip))
	# Resultado: ¿Es VIP? true

Ejemplo completo en contexto de juego

Resumen de jugador
extends Node

var nombre = "Elena"
var vida = 85
var ataque = 22
var tiene_escudo = true

func _ready() -> void:
	print("=== Resumen ===")
	print("Nombre: " + nombre)
	print("Vida: " + str(vida))
	print("Ataque: " + str(ataque))
	print("Escudo: " + str(tiene_escudo))

Alternativa: usar comas en print()

print() también acepta varios valores separados por comas. No necesitas str() porque print convierte automáticamente. La diferencia es que agrega un espacio entre cada valor.

Comas vs concatenación
extends Node

func _ready() -> void:
	var vidas = 3
	var nivel = 5

	# Con comas (agrega espacios automáticamente)
	print("Vidas:", vidas, "Nivel:", nivel)
	# Resultado: Vidas: 3 Nivel: 5

	# Con concatenación (tú controlas los espacios)
	print("Vidas: " + str(vidas) + " | Nivel: " + str(nivel))
	# Resultado: Vidas: 3 | Nivel: 5
✓
Regla de oro
Si usas + para unir textos, todo lo que unas debe ser String. Si algo no es texto (número, booleano), envuélvelo con str().

"Daño: " + str(20) → ✅ Correcto
"Daño: " + 20 → ❌ Error

5) Condicionales (if / elif / else)

Los condicionales permiten tomar decisiones. El programa evalúa una condición y ejecuta un bloque u otro.

Estructura básica

Plantilla
func _ready() -> void:
	# Reemplaza "condicion" por algo real (ej: vidas > 0)
	if condicion:
		# código si es verdadero
	elif otra_condicion:
		# otra opción
	else:
		# si nada se cumplió

6) Ciclos (for / while)

Un ciclo repite un bloque de código. En juegos es común repetir: contar, iterar enemigos, revisar listas, temporizadores.

for (cuando sabes cuántas veces)

Contar 0 a 4
extends Node

func _ready() -> void:
	for i in range(5):
		print(i)

Nota: range(5) genera 0,1,2,3,4.

while (cuando depende de una condición)

Repetir hasta cumplir condición
extends Node

func _ready() -> void:
	var contador = 0

	while contador < 3:
		print("Intento: ", contador)
		contador += 1
Error común: ciclo infinito peligro
Un while puede volverse infinito si la condición nunca cambia.

❌ Infinito

var x = 0
while x < 5:
	print(x)
	# falta x += 1

✅ Correcto

var x = 0
while x < 5:
	print(x)
	x += 1
!
Antes de los ejercicios: ¿qué es un array?
Algunos ejercicios usan arrays (listas). Un array es simplemente una lista ordenada de valores dentro de corchetes:

var pociones = ["Vida", "Maná", "Fuerza"]

Puedes acceder a cada elemento por su posición (empieza en 0):
pociones[0] → "Vida" · pociones[1] → "Maná" · pociones[2] → "Fuerza"

Para saber cuántos elementos tiene, usa .size():
pociones.size() → 3

No te preocupes por aprenderlo todo ahora, los arrays se explican a fondo en la sección 8. Por ahora solo necesitas saber esto para resolver los ejercicios.

8.8 · Arrays bidimensionales (matrices)

En Godot Engine (incluyendo Godot 4.x) no existe un tipo especial llamado "matriz" para datos generales, como en algunos lenguajes. Pero sí puedes crear matrices fácilmente usando arrays dentro de arrays. En otras palabras: una matriz es un Array que contiene otros Array.

Ejemplo de matriz en GDScript
extends Node

var matriz = [
	[1, 2, 3],
	[4, 5, 6],
	[7, 8, 9]
]

func _ready() -> void:
	print(matriz[0])   # [1, 2, 3]  — primera fila
	print(matriz[1][2])  # 6       — fila 1, columna 2

Esto visualmente sería:

1 2 3
4 5 6
7 8 9

Cada fila es un Array. Para acceder: matriz[fila][columna].

Recorrer una matriz

Usa un for anidado: el externo recorre las filas, el interno las columnas.

for anidado
extends Node

var matriz = [[1, 2], [3, 4], [5, 6]]

func _ready() -> void:
	for fila in matriz:
		var linea = ""
		for celda in fila:
			linea += str(celda) + " "
		print(linea)

Acceder con índices

Si necesitas fila y columna explícitas, usa range().

Acceso por índices
extends Node

var tablero = [["A","B"], ["C","D"]]

func _ready() -> void:
	for i in range(tablero.size()):
		for j in range(tablero[i].size()):
			print("Fila ", i, ", Col ", j, ": ", tablero[i][j])
!
Filas de distinto tamaño
En GDScript las filas pueden tener distintas longitudes. Para tableros regulares, asegúrate de crear filas del mismo tamaño.

8.9 · Arrays de más dimensiones

Puedes anidar arrays tantas veces como necesites. Un array tridimensional sería un array de matrices; uno de cuatro dimensiones, un array de "cubos" de datos. En juegos se usan para: grillas 3D, niveles con pisos, mazmorras por coordenadas (x, y, piso), datos temporales (ej: guardar estados de varios frames).

Ejemplo 3D: piso → fila → columna
extends Node

# 2 pisos, cada uno con una matriz 2x2
var mazmorra = [
	[[1, 2], [3, 4]],   # Piso 0
	[[5, 6], [7, 8]]    # Piso 1
]

func _ready() -> void:
	# Acceder: piso 1, fila 0, columna 1
	print(mazmorra[1][0][1])  # 6

4D y más: sigue el mismo patrón. datos[i][j][k][l] para cuatro dimensiones. La lógica es la misma: cada nivel de corchetes recorre un eje.

Error común: índice fuera de rango típico

Si intentas acceder a un índice que no existe, Godot lanzará un error.

❌ Error

var armas = ["Daga", "Espada"]
print(armas[5])
# ❌ Error: Index 5 is out of bounds (size: 2)

✅ Correcto

var armas = ["Daga", "Espada"]
# Verificar antes de acceder
if armas.size() > 5:
	print(armas[5])
else:
	print("Ese índice no existe")
!
Recuerda
Un array de 3 elementos tiene índices 0, 1 y 2. El índice 3 ya no existe. El último índice siempre es .size() - 1.

10) Clases en Godot (cada script es una clase)

En Godot, un script define el comportamiento de un nodo. Técnicamente, cada script es una clase. "Clase" = plantilla. "Objeto" = instancia real en el juego (un enemigo específico, un jugador, etc.).

extends — de dónde hereda el script

extends CharacterBody2D significa: "este script es un tipo de CharacterBody2D". Así ganas movimiento, colisiones y métodos útiles.

Ejemplo simple

	velocity.x = velocidad
	move_and_slide()

Variables y funciones como "atributos" y "métodos"

Una clase guarda datos (variables) y acciones (funciones). Eso describe al objeto.

Daño
extends Node

var vidas = 3

	vidas -= cantidad
	if vidas <= 0:
		print("Muerto")
Conceptos Godot que conviene nombrar desde ya
  • Nodo: pieza del juego (sprite, colisión, audio, etc.)
  • Escena: árbol de nodos guardado como archivo
  • Señales: eventos (ej: botón presionado)
  • _ready / _process / _physics_process: ciclo de vida

11) Errores de sintaxis típicos (y cómo leerlos)

Un programador real no evita errores: aprende a leerlos. La meta es que puedas: (1) interpretar el mensaje, (2) ubicar la línea, (3) corregir.

✓
Regla práctica
El error suele estar en la línea marcada… o una línea antes (por ejemplo, un : faltante).
🔴 Bloquea Instrucción ejecutable en el cuerpo del script

Godot dice: Unexpected identifier "print" in class body

❌ Con error

extends Node

var vidas = 3

print(vidas)  # ❌ NO se permite aquí

✅ Corregido

extends Node

var vidas = 3

func _ready() -> void:
	print(vidas)  # ✅ dentro de función
¿Cómo lo identificas? Hay una línea que "hace algo" (print, if, for) al mismo nivel que var y func.
Solución: Mueve esa instrucción dentro de una función como _ready(). Fuera solo van declaraciones: var, const, func, @export, @onready.
🔴 Bloquea Falta indentación

Godot dice: Expected indented block

❌ Con error

if x > 0:
print(x)     # ❌ falta sangría

✅ Corregido

if x > 0:
	print(x)  # ✅ con tab/espacios
¿Cómo lo identificas? La línea debajo del if/for/func no tiene sangría.
Solución: Indenta con tab o espacios de forma consistente.
🔴 Bloquea Falta ":"

Godot dice: Expected ':'

❌ Con error

if x > 0     # ❌ falta :

✅ Corregido

if x > 0:    # ✅ con :
	print(x)
¿Cómo lo identificas? Revisa el final de la línea del if, for, while o func.
Solución: Agrega : al final de la línea.
🔴 Bloquea Paréntesis sin cerrar

Godot dice: Expected ')'

❌ Con error

print("Hola"   # ❌ falta )

✅ Corregido

print("Hola")  # ✅ cerrado
¿Cómo lo identificas? Se marca el final de la línea.
Solución: Cuenta los paréntesis abiertos y ciérralos todos.
🔴 Bloquea Comillas sin cerrar

Godot dice: Unterminated string

❌ Con error

var s = "Hola  # ❌ falta "

✅ Corregido

var s = "Hola" # ✅ cerrado
¿Cómo lo identificas? El texto queda resaltado de un color raro o todo rojo desde esa línea en adelante.
Solución: Cierra las comillas. Asegúrate de que cada " tenga su par.
🟠 Confunde Variable no declarada

Godot dice: Identifier not found

❌ Con error

print(nombre)  # ❌ nunca se declaró

✅ Corregido

var nombre = "Mario"
print(nombre)  # ✅ declarada antes
¿Cómo lo identificas? Aparece subrayado rojo en el nombre de la variable.
Solución: Declara la variable con var antes de usarla, o revisa si la escribiste mal.
🟠 Confunde Usar = en vez de ==

Godot dice: Invalid assignment in expression

❌ Con error

if x = 3:      # ❌ esto asigna

✅ Corregido

if x == 3:     # ✅ esto compara
	print("x vale 3")
¿Cómo lo identificas? Dentro de un if o while no se puede asignar.
Solución: Usa == (doble igual) para comparar.
🟠 Confunde Mezclar tabs y espacios

Godot dice: Inconsistent use of tabs and spaces

❌ Con error

if x > 0:
	print("OK")    # con tab
    print("Sigue") # con espacios ❌

✅ Corregido

if x > 0:
	print("OK")    # todo con tab ✅
	print("Sigue") # todo con tab ✅
¿Cómo lo identificas? Parece indentado correctamente pero Godot marca error.
Solución: Usa solo tabs o solo espacios en todo el archivo. En Godot, tabs es lo estándar.
🟢 Lógico Olvidar paréntesis al llamar función

Godot: no siempre da error, pero la función no se ejecuta.

❌ Con error

saludar   # ❌ no se ejecuta

✅ Corregido

saludar() # ✅ se ejecuta
¿Cómo lo identificas? No pasa nada al ejecutar, no hay error visible.
Solución: Siempre pon () al llamar una función, aunque no tenga parámetros.
🟢 Lógico Índice fuera de rango

Godot dice: Index out of bounds

❌ Con error

# Array tiene 3 elementos (0, 1, 2)
enemigos[10]  # ❌ no existe

✅ Corregido

if 10 < enemigos.size():
	print(enemigos[10]) # ✅ validado
¿Cómo lo identificas? El error aparece al ejecutar, no al escribir.
Solución: Valida con .size() antes de acceder, o usa índices que sepas que existen.
!
Método de 4 pasos para leer errores
1. Lee el mensaje (¿esperaba ":"?, ¿bloque indentado?, ¿identificador?).
2. Ve a la línea marcada (pero revisa también la línea anterior).
3. Busca patrones: falta ":", comillas, paréntesis, indentación.
4. Corrige y vuelve a ejecutar (uno a la vez).
